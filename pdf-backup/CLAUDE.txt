WHAT THIS IS

A system that automates the graded weekly performance assessments that Matt and
Claude have been doing manually since February 2026. It pulls data from multiple
sources (Apple Health, Gmail, GitHub, Google Calendar, Claude conversation history),
scores each category using a defined rubric, compares period-over-period, and delivers
the report via an iOS app with an Apple Watch companion.

This is NOT a generic fitness tracker or productivity app. It is a deeply personalized
performance assessment engine built around Matt's specific life situation, business
(Ridge Cell Repair LLC), technical projects, and health goals.

D-, F

Grading Algorithm

1. Collect data from all sources (HealthKit, Gmail, GitHub, Calendar, manual input)
2. Compute metrics per category (see rubric above)
3. Compare to baselines and prior period
4. Apply rubric to assign letter grade per category
5. Compute weighted average for overall grade
6. Generate recommendations based on lowest-scoring categories and trend

     direction
7. Store assessment in local database
8. Push notification to iOS/Watch

Health Grading Specifics

func gradeHealth(metrics: HealthMetrics, baseline: HealthBaseline) -> Grade {
       var score: Double = 0.0

// HRV (most important -- weight 30%)

if metrics.hrv >= 60 { score += 1.3 }  // Good

else if metrics.hrv >= 50 { score += 0.9 } // Acceptable

else if metrics.hrv >= 40 { score += 0.4 } // Concerning

else { score += 0.0 }                  // Crisis

// Resting HR (weight 20%)
if metrics.restingHR <= 65 { score += 0.86 }
else if metrics.restingHR <= 70 { score += 0.6 }
else if metrics.restingHR <= 75 { score += 0.3 }
else { score += 0.0 }

// Steps (weight 15%)
if metrics.stepsPerDay >= 7500 { score += 0.65 }
else if metrics.stepsPerDay >= 5000 { score += 0.4 }
else { score += 0.15 }

// Exercise Minutes (weight 15%)
if metrics.exerciseMinPerWeek >= 150 { score += 0.65 }
else if metrics.exerciseMinPerWeek >= 75 { score += 0.4 }
else if metrics.exerciseMinPerWeek >= 30 { score += 0.2 }
else { score += 0.0 }

// Meal plan adherence (weight 10%)
if metrics.mealPlanDaysFollowed >= 6 { score += 0.43 }
else if metrics.mealPlanDaysFollowed >= 4 { score += 0.25 }
else { score += 0.0 }

// Trend bonus/penalty (weight 10%)
           if metrics.hrv > baseline.hrv * 1.1 { score += 0.43 }
           else if metrics.hrv < baseline.hrv * 0.9 { score -= 0.2 }

           return gradeFromScore(score, maxScore: 4.3)
    }

DON'TS

 1. DO NOT use a cloud backend for health data. HealthKit data stays on-device. Period.
 2. DO NOT grade Fiverr activity as pipeline -- it's 100% scammers.
 3. DO NOT dismiss the Kalshi trader as a distraction. It's the highest-ROI asset.
 4. DO NOT penalize private repos. Matt's best work (Wraith, ClaudioOS, Kalshi) is

      private for valid IP protection reasons.
 5. DO NOT use React Native or Flutter. This is a native Swift/SwiftUI app.
 6. DO NOT require internet for basic functionality. Assessment history and health data

      should work offline.
 7. DO NOT auto-share data anywhere. Matt controls what leaves the device.
 8. DO NOT use localStorage in any web views -- use SwiftData.
 9. DO NOT count the same application twice. Greenhouse sends security codes +

      confirmations, which creates duplicate emails per application. Deduplicate by
      company+role.
10. DO NOT assume mmichels88@gmail.com data is accessible -- it's not connected to
      OAuth.
